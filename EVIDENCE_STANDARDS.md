# BALI EVIDENCE STANDARDS

## Purpose

Bali exists to discover evidence-backed crypto strategies. Evidence must be searchable, repeatable, and comparable.

## Minimum evidence for a strategy test

Every serious test should record:

- run ID
- strategy name
- strategy version/hash
- dataset source
- coin/pair
- timeframe
- date range
- fees
- slippage
- spread assumptions
- in-sample result
- walk-forward result
- out-of-sample result
- drawdown
- profit factor
- Sharpe/Sortino if available
- win rate
- trade count
- rejection reason if failed
- report file location

## Proof gates

A strategy cannot become a paper champion candidate until it survives:

- clean data gate
- in-sample test
- walk-forward test
- out-of-sample test
- stress/slippage test
- drawdown limit
- minimum trade count
- paper/simulation forward monitoring

## Profit claims

No profitability claims are allowed without evidence. A backtest result is a test result, not proof of future profit.
